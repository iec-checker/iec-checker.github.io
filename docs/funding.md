# Sponsor Development

iec-checker is an open-source tool available for free.

Here are examples of what could be done on a funded basis.

### Vendored extensions support

Since industrial automation development tools don't have a single standard and almost every serious toolchain introduces its own extensions or semantic changes in the IEC languages, it is not feasible to support all of these out of the box. Funded work can add first-class support for your vendor's dialect.

### More input formats

SEL XML support was added this way — other input formats used by PLC programming environments can be supported as well.

### More languages

IL, FBD, LD, and SFC could be supported. Since these languages don't have a stable *open-source* compiler and all the commercial tools keep their implementations proprietary, they were not implemented — but it can be done for your tooling.

Additional non-standard DSLs used in control programs may be supported as well.

### More rules

More PLCOpen rules or rules from other standards can be implemented. While the tool does not have certification, it may be used in a safety qualification process.
